  Community

Lots of ways to get involved with XboxLeaders

  • Follow @xboxleaders
  • Tweet

XboxLeaders has a vibrant community of folks helping each other out. You can get support, report bugs, request new features, submit pull requests, and check upcoming milestones.

Getting Support

Having trouble getting the API up and running? Something not working the way you think it should? I hate that I don't have time to answer support emails anymore. So here are some things you might wanna do:

  1. Make sure you've read the latest version of how to get started. It's been updated recently to make the API even easier to use.
  2. Check out the existing questions tagged as XboxLeaders over on Stack Overflow. Other folks might have had the same question you've had.
  3. Can't find the answer to your question on Stack Overflow? Ask a new question, then send me an email and I might be able to take a look.

Reporting Bugs

Found a problem with the API? Feel free to submit an issue on the GitHub project. But please keep the following in mind:

  1. Please be nice. XboxLeaders is a happy place.
  2. Please search to see if your icon request already exists.
  3. Before opening any issue, please read the generic issue guidelines, by Nicolas Gallagher.
  4. After doing everything above, feel free to submit an issue.

Requesting New Features

New icons mostly start as requests by the XboxLeaders community on GitHub. Want to request a new feature? Here are some things to keep in mind:

  1. Please be nice. XboxLeaders is a happy place.
  2. Please search to see if your feature request already exists.
  3. Please and thank you if you include the following:
    • Title your new issue Feature request: Feature Name.
    • Include a few use cases for your requested feature. How do you plan on using it?
    • Attach an paragraph or two that represent the idea you're going for.

Submitting Pull Requests

Found a way to solve a bug in the API? Want to contribute new features? Here are a few things to remember:
  1. Please submit all pull requests against *-wip branches.
  2. All pull requests submitted against master will be sumarily closed and this guide referenced.
  3. After doing everything above, feel free to submit a pull request.

Project Milestones

Want to keep up with what's planned for the API? Check out our milestones on the GitHub project.
